## Module Requirements
### **Functional Requirements**
  - Check that users can log in using both basic (username/password) and social account options.
  - Review if users can successfully reset or change their passwords through guided steps.
  - Confirm that reset links or codes work only once and within a limited time.
  - Verify that all messages and prompts during login, reset, or password update are clear and easy to understand.
### **Security Requirements**
  - Validate if password rules (such as required length and mix of characters) are being followed.
  - Check that passwords are stored securely in the system.
  - Ensure only logged-in users can change their password or account details.
  - Test if reset links or codes expire correctly and cannot be reused.
  - Confirm that there are controls in place to stop repeated or suspicious login attempts.
### **Performance Requirements**
  - Check how long it takes to log in or complete a password reset.
  - Review the system's behavior when multiple users try to log in at the same time.
  - Make sure login and password processes are quick and donâ€™t cause long delays.
### **Accessibility & Responsiveness**
  - Confirm that all login and password forms are easy to use with screen readers and keyboards.
  - Test the layout and behavior on both desktop and mobile devices.
  - Ensure users get proper feedback when they enter incorrect information or complete steps.
